# Структура папок для изображений

## Основная структура:

```
public/images/
├── carousel/          # Главная карусель (6 изображений)
├── how-it-works/      # Блок "Как это работает" (1 изображение)
└── products/          # Галереи продуктов
    ├── product-1/     # Продукт 1 (5 изображений)
    ├── product-2/     # Продукт 2 (5 изображений)
    └── product-3/     # Продукт 3 (5 изображений)
```

## Требования к изображениям:

### 1. Главная карусель (`carousel/`)
- **Количество:** 6 изображений
- **Имена файлов:** `1.webp`, `2.webp`, `3.webp`, `4.webp`, `5.webp`, `6.webp`
- **Форматы:** .webp, .png или .jpg
- **Соотношение сторон:** 3:4 (вертикальные)

### 2. Блок "Как это работает" (`how-it-works/`)
- **Количество:** 1 изображение
- **Имя файла:** `1.webp`
- **Форматы:** .webp, .png или .jpg
- **Соотношение сторон:** 3:4 (вертикальное)

### 3. Галереи продуктов (`products/product-1/`, `products/product-2/`, `products/product-3/`)
- **Количество:** по 5 изображений в каждой папке
- **Имена файлов:** `1.webp`, `2.webp`, `3.webp`, `4.webp`, `5.webp`
- **Форматы:** .webp, .png или .jpg
- **Соотношение сторон:** 3:4 (вертикальные)

## Примеры имен файлов:

✅ Правильно:
- `1.webp`, `2.png`, `3.jpg`
- `1.webp`, `1.png`, `1.jpg` (система попробует все форматы)

❌ Неправильно:
- `image1.webp`, `photo-1.jpg` (неправильное имя)
- `01.webp`, `001.jpg` (неправильное имя)

## Примечание:

Система автоматически пытается загрузить изображения с расширениями в следующем порядке:
1. `.webp` (предпочтительно)
2. `.png`
3. `.jpg`

Если изображение не найдено, будет показан плейсхолдер с номером изображения.
